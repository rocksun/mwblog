亚马逊已向其数万名开发者提供了对 Anthropic 的 Claude Code 的即时访问权限，并且很快将开放对 OpenAI 的 Codex 的访问，将其智能体编程工具（Agentic coding tools）的范围扩展到了自家的 Kiro 之外。

在一份由 [*Business Insider*](https://www.businessinsider.com/amazon-claude-code-codex-all-employees-after-pushback-2026-5) 获取的内部员工通知中，亚马逊软件构建者体验（Software Builder Experience）副总裁 Jim Haughwout 表示，这两款工具都将在 Amazon Web Services（AWS）和 Amazon Bedrock 上运行，后者是亚马逊用于构建和扩展生成式 AI 应用程序的全托管服务。

此举将允许这家电子商务巨头在为开发者提供顶级编程助手的向时，避免复杂的基础设施设置，简化容量管理，并严格控制数据安全和合规性。

## 开发者请注意：有求必应

Haughwout 在给亚马逊员工的通知中表示，这家科技公司将于 5 月 12 日开放 Claude Code 和 Codex 的访问权限，旨在最终更好地服务客户：“为了帮助你们为客户进行更多创新，我们正在扩大你们可以使用的智能体 AI 工具。”

但看起来亚马逊最终可能是在听到员工对第三方工具限制以及公司强推自研工具 Kiro 的抱怨后，向开发者的需求妥协了。

[Kiro](https://kiro.dev/blog/introducing-kiro/) 于 2025 年推出，是基于 Bedrock 构建的智能体编程服务。据 [*Business Insider*](https://www.businessinsider.com/amazon-engineers-grate-against-internal-limits-claude-code-kiro-ai-2026-2) 2 月份报道，亚马逊的内部消息曾要求团队在生产代码中优先使用 Kiro，而非其他未经批准的第三方工具（其中包括 Claude Code）。但亚马逊员工迅速做出了反击，据报道，一个讨论帖包含了约 1,500 名员工的联名支持，敦促公司正式采纳 Claude Code。

仅仅几个月后，这一批准就下来了。

Claude Code 以及即将上线的 Codex 现在已被批准用于生产环境，员工不再需要特殊许可即可使用这些工具。

尽管如此，正如一位亚马逊发言人告诉 *Business Insider* 的那样，Kiro 依然存在：“我们的构建者正在使用 Kiro 进行智能体编程，现在随着 Claude Code 和 Codex 都在 AWS 上运行，我们也提供了额外的工具。”

## 这对电子商务开发意味着什么

通过在全公司范围内提供 Claude Code 和 Codex，亚马逊正在显著扩大其对第三方编程工具的使用。

> “现在的约束需要转向审查、验证，以及确保系统行为符合你的预期。”

当被问及这对这家科技公司的开发者意味着什么时，Galileo 的主任软件工程师 Satyam Dhar（曾任职于 Adobe 和 Amazon）告诉 *The New Stack*，他预计最大的影响将体现在内部工作流上：“现在的约束需要转向审查、验证，以及确保系统行为符合你的预期。” 同时，他预测领导层将“不得不重新思考他们评估生产力和产出的方式”。

亚马逊对 Claude Code 和 Codex 的新批准标志着电子商务领域向 AI 和智能体主导的工作模式的持续转变。

例如，在去年春天的一篇[博客文章](https://corporate.walmart.com/news/2025/05/29/inside-walmarts-strategy-for-building-an-agentic-future#:~:text=Key%20Insights,Innovation)中，沃尔玛详细介绍了其自身专用智能体 AI 工具的开发情况。

这家零售商强调了其对智能体 AI 的“手术刀式”精准方法，称其优先开发针对零售特定任务的智能体，而非通用案例。其 AI 驱动的购物助手 Customer Assistant 和 Trend-to-Product 只是该零售商引用的“智能体 AI 的早期成功和重大影响”的几个例子。

亚马逊在 AI 驱动的电子商务领域也已经有了类似的尝试，例如 2024 年推出的 AI 购物助手 [Rufus](https://www.aboutamazon.com/news/retail/amazon-rufus)。但通过批准全公司使用 Claude Code 和 Codex，它正赋予开发者更广泛的工具访问权限，这可能有助于其更快地开发和发布更多新功能。

就沃尔玛而言，它对 AI 编程辅助并不陌生。据 [*CIO Dive*](https://www.ciodive.com/news/Walmart-generative-AI-agents-coding-tool/740545/) 报道，在 2025 财年第四季度财报电话会议上，该零售商表示 AI 驱动的编程辅助帮助节省了 400 万个开发小时。沃尔玛全球技术平台执行副总裁 Sravana Karnati 去年年底告诉 [*《华尔街日报》*](https://www.wsj.com/articles/walmart-wants-more-developers-and-more-ai-agents-to-automate-their-work-e4afaa22)，超过 95% 的沃尔玛工程师已经在使用 AI 编程助手工具。

## AI 编程将长期存在的又一信号

当被问及亚马逊的举动将如何塑造软件开发时，Satyam Dhar 认为这是一个信号，表明 AI 编程正在超越实验阶段，开始成为生产常态：

“它正在改变工程师投入时间的地方。减少了逐行编写代码的时间，而将更多时间花在定义结构、审查输出和思考系统行为上。”

亚马逊的这一更新发布之际，该公司正继续深化与 Anthropic 和 OpenAI 的联系。

今年 4 月，亚马逊和 Anthropic 宣布了一项[协议](https://thenewstack.io/anthropic-amazon-aws-investment/)，根据该协议，Anthropic 将在 AWS 上提供 Claude 平台，接受亚马逊高达 250 亿美元的投资，并承诺在 10 年内向 AWS 技术投资 1000 亿美元，以确保获得高达 5 吉瓦（GW）的亚马逊 Trainium 和 Graviton 核心算力。

今年早些时候，亚马逊与 OpenAI 签署了一项[协议](https://www.aboutamazon.com/news/aws/amazon-open-ai-strategic-partnership-investment)，向这家 AI 公司投资高达 500 亿美元，双方承诺共同创建一个由 OpenAI 模型驱动的“有状态运行时环境（Stateful Runtime Environment）”，并可在 Amazon Bedrock 上供 AWS 客户使用。
