OpenAI 为 Codex 推出 Sites、Annotations 及多款新插件，旨在服务知识工作者。用户现可创建交互网页、针对性修改文档，并利用专用插件提升工作效率。

> 译自：[OpenAI's Codex adds new tools — Sites, Annotations, more plugins — for knowledge workers](https://thenewstack.io/openai-codex-knowledge-workers/)
> 
> 作者：Frederic Lardinois

正如 OpenAI 周二[宣布](https://openai.com/index/codex-for-knowledge-work/)的那样，在 Codex 每周 500 万活跃用户中，现在有 20% 是知识工作者而非程序员。这里有一个巨大的市场亟待争夺——Anthropic 也在通过 Cowork 等功能争夺这一市场——因此，OpenAI 现在调整 Codex 以适应这群用户也就不足为奇了。

同样在周二，该公司宣布了两个新功能——Sites 和 Annotations——以及一套专门针对这些用户的插件。

## OpenAI 推出 Sites

在这些新功能中，最重要的是 Sites。顾名思义，这不仅能让 Codex 用户创建交互式网站，还能通过 URL 将其分享给工作空间中的任何人。

“团队无需为了适应单一工具或文件的局限而调整工作，而是可以创建适合该工作的网站，”OpenAI 在其公告中表示。

这一创意的初衷是允许用户创建自定义仪表盘、网站和应用程序。例如，承载基于财务模型的场景规划器，或者作为托管新产品发布材料的场所，并实时更新消息、里程碑和决策。

> “团队无需为了适应单一工具或文件的局限而调整工作，而是可以创建适合该工作的网站。”

正如 OpenAI 所描述的，“这些更新指向了一种不同的职场 AI 模式：它不仅仅是一个回答问题的聊天机器人，也不是一个专注于单一体验的智能体，而是一个协作空间。在这里，团队可以引入特定角色的上下文，创建工作成果，当场检查并改进输出，并将其转化为其他人可以使用的交互式工具。”

为了扩展这一功能，OpenAI 正在与 Wix、Base44、Replit、Lovable、Figma 和 Emergent 等合作伙伴合作，利用他们各自的能力来增强这些网站。

商业和企业团队将能够在 Codex 应用中使用 Sites。

## Annotations（批注）

另一个新功能扩展了 Annotations（批注）。今年早些时候，当 OpenAI 开始将 Codex 预览为公司的桌面超级应用时，就已经为开发人员推出了该功能。有了它，Codex 用户可以将他们的智能体指向文档、幻灯片、电子表格或网站的特定部分，并让其对该部分进行更改（或者将该选定部分用作其他内容的上下文）。

这之所以成为可能，是因为在最近的一些更新中，Codex 现在可以直接在应用中打开这些文档。

例如，开发人员以前就已经能够对代码和 Markdown 文件执行此操作，而此次更新现在也向知识工作者及其通常处理的文档开放了这一功能。

## 更多插件

最后，OpenAI 还将推出一套新插件。这些是应用程序、技能和集成的捆绑包，每个新插件都迎合特定类型的工作。目前，所有这些都由 OpenAI 亲自挑选，但计划是让该公司的合作伙伴能够直接在 Codex 和 ChatGPT 中提供他们自己的插件。

新插件包括一个数据分析插件，可以处理来自 Snowflake、Databricks、Hex 和 Tableau 等工具的数据；而新的销售插件可以帮助销售团队使用来自 Salesforce、HubSpot、Slack 和其他工具的数据为客户会议做准备。此外还有用于产品设计和投资银行业的插件。

![](https://cdn.thenewstack.io/media/2026/06/7b3d1492-public-equity-investing-plugin-post-earnings-deep-dive.gif)

图片来源：OpenAI。

即将登陆 Codex 的还有用于企业财务、私募股权投资、营销策略、战略咨询和法律的插件。

这个垂直领域列表听起来与 Anthropic 今年早些时候[为 Claude Cowork 发布](https://github.com/anthropics/knowledge-work-plugins)的插件集合非常相似，不过，这些也正是目前 AI 智能体可以发挥其最佳作用的以文档为中心的垂直领域。
