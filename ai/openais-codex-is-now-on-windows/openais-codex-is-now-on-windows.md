OpenAI 的 Codex 智能编程应用现已登陆 Windows，备受期待。它专为 Windows 环境打造，提供原生支持和智能体管理功能，使用 GPT-5.3-Codex 模型，面向所有 ChatGPT 用户。

> 译自：[OpenAI's Codex is now on Windows](https://thenewstack.io/openais-codex-is-now-on-windows/)
> 
> 作者：Frederic Lardinois

OpenAI 的 Codex 智能编程应用现已[在 Windows 上可用](https://apps.microsoft.com/detail/9plm9xgg6vks?hl=en-US&gl=US)。

称 Codex 在 OpenAI 取得了巨大成功，这都算是轻描淡写。适用于 Mac 的 Codex 应用于[2月初发布](https://thenewstack.io/openais-codex-desktop-app-is-all-about-managing-agents/)，仅在第一周就被下载了超过100万次，目前每周活跃用户已达160万。

Windows 版本也可能需求量很大：OpenAI 表示有超过50万名开发者在等候名单上。

正如 OpenAI 强调的，Windows 版本不仅仅是为了与微软操作系统兼容而构建，而是“为真正的 Windows 开发环境”而设计，正如一位 OpenAI 发言人在给《The New Stack》的电子邮件中所说。

该应用旨在提供原生沙盒和工作流，以便 Windows 开发者可以使用他们已经熟悉的工具。默认情况下，该应用使用其原生的 Windows 沙盒，但也有一个选项可以使用适用于 Linux 的 Windows 子系统及其工具。

如果你选择 Windows 原生方式，Windows 版 Codex 会使用操作系统级别的控制，例如受限令牌和文件系统访问控制，确保智能体可以在 PowerShell（微软 Windows 的默认 shell）等环境中运行。

![](https://cdn.thenewstack.io/media/2026/03/39a606d9-02@2x-scaled.png)

OpenAI 的 Windows 版 Codex 应用（图片来源：OpenAI）。

在大多数方面，Windows 版本的外观和感觉与 Mac 版本几乎完全相同。相同的技能、自动化和对工作树的支持都可在 Windows 上使用。还有一些 Windows 特有的技能，包括为编写 Windows 应用的开发者提供的 WinUI 技能。

> 还有一些 Windows 特有的技能，包括为编写 Windows 应用的开发者提供的 WinUI 技能。

Codex 始终与众不同的一点是，它更侧重于管理智能体而非代码本身。你可以随时根据需要查看差异并切换到你喜欢的 IDE，但默认视图专注于你与智能体的交互。OpenAI 将其描述为“一种为智能体设计的全新指挥中心”，这种说法感觉很贴切。

![OpenAI 的 Windows 版 Codex 应用。技能菜单。](https://cdn.thenewstack.io/media/2026/03/6baf7800-03@2x-scaled.png)

Codex 默认使用的模型是 OpenAI 最近推出的编码专用 [GPT-5.3-Codex](https://thenewstack.io/openais-gpt-5-3-codex-helped-build-itself/) 模型，并可切换到 GPT-5.2-Codex、GPT-5.1-Codex-Max、GPT-5.2，以及用于需要快速运行的任务的 GPT-5.1-Codex-Mini。用户还可以设置每个模型的推理级别。

Windows 版 Codex 现已面向所有 ChatGPT Free、Go、Plus、Pro、Business、Enterprise 和 Edu 用户开放。
