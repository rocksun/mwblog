[GitHub](https://github.com/) 订阅者[现在](https://github.blog/changelog/2026-02-04-claude-and-codex-are-now-available-in-public-preview-on-github/)可以[选择编码代理](https://thenewstack.io/github-embraces-the-coding-agent-competition-with-agent-hq/)来帮助他们进行创作。

除了[GitHub 自家的 Copilot](https://thenewstack.io/github-copilot-wants-to-become-your-peer-programmer/)，用户还可以通过 GitHub 推出的一项名为 Agent HQ 的新功能，在 GitHub 和 VS Code 中选择 [Anthropic 的 Claude Code](https://thenewstack.io/anthropics-claude-code-comes-to-web-and-mobile/)、[OpenAI 的 Codex](https://thenewstack.io/openais-codex-desktop-app-is-all-about-managing-agents/) 以及自定义代理。

Agent HQ 目前对 Copilot Pro+ 和企业版订阅者开放，但该公司计划很快将 Claude 和 Codex 的访问权限扩展到其他 Copilot 订阅层级。

该公司最初在 2025 年 10 月的 [GitHub Universe 活动](https://thenewstack.io/github-embraces-the-coding-agent-competition-with-agent-hq/)上宣布了这项新功能，并计划在当年晚些时候推出（尽管时间表显然有所调整）。

所有这些软件开发代理都已以某种形式与 GitHub 集成并使用 GitHub。但通过 Agent HQ，开发人员现在可以从一个统一的仪表板与所有这些代理协同工作，并且可以根据不同的任务需求在它们之间进行切换。

另一个有趣的用例是，将相同的任务分配给不同的代理，以了解它们如何对给定问题进行推理，以及它们可能提供哪些解决方案。

正如 GitHub 首席运营官 [Kyle Daigle](https://www.linkedin.com/in/kyledaigle/) 去年在 Agent HQ 发布时告诉 *The New Stack* 的那样，公司希望为开发人员提供选择，无论是他们想使用的 IDE 还是他们更喜欢合作的代理。

Daigle 当时表示：“我们 Agent HQ 的目标是，拥有一个单一的地方，你可以在其中使用基本上任何想要集成的编码代理，并拥有一个单一的玻璃面板——一个任务控制界面，我可以在其中看到所有任务，它们正在做什么，它们处于何种代码状态——比如创建、代码审查等，并向所有其他编码代理提供我们构建 GitHub Copilot 编码代理所使用的底层原语。”

开发人员可以将任何任务分配给他们偏好的代理，让其异步工作。Claude 和 Codex 会话可以从 GitHub.com、GitHub 移动应用程序和 VS Code 中启动。开发人员可以将问题分配给这些代理，并且代理可以提交草稿拉取请求以供审查。

代理还可以处理现有的拉取请求，并且正如已经成为标准的那样，在 PR 评论中提及 `@Copilot`、`@Claude` 或 `@Codex` 也会启动任何后续工作。

GitHub 指出，这些代理所做的所有工作都将记录下来以供将来审查，“因此它们的输出自然地符合您已经使用的评估开发人员贡献的相同工作流程。”

每次会话将消耗一个高级请求。这些[高级请求](https://docs.github.com/en/billing/concepts/product-billing/github-copilot-premium-requests)是 GitHub 用于使用更高级功能和高级模型的“货币”。Copilot Pro+ 用户获得 1,500 个此类请求，企业用户获得 1,000 个，每个额外请求的成本为 0.04 美元。
