微软与 OpenAI 修订协议，打破了 Azure 的排他性。OpenAI 将接入 AWS 等平台以拓展市场，微软则可自由合作 Anthropic 等对手。此举标志着 AI 领域从排他性结盟转向灵活、多云的新生态。

> 译自：[Microsoft-OpenAI rewrite opens the door for Anthropic and Google](https://thenewstack.io/openai-microsoft-partnership-restructured/)
> 
> 作者：Adrian Bridgwater

ChatGPT 创建者 [OpenAI](https://thenewstack.io/openai-aims-to-make-chatgpt-the-operating-system-of-the-future/) 和微软 [于周一宣布](https://openai.com/index/next-phase-of-microsoft-partnership/) 重新修订其联合伙伴关系协议。两家公司的重新定位可能为建立一个全新的、更加分散的联盟铺平道路。

“今天，我们宣布了一项修订协议，旨在简化我们的伙伴关系以及我们的协作方式，该协议基于灵活性、确定性，并专注于广泛提供 AI 的效益，”OpenAI 表示。

OpenAI 最终将能够在 Azure 之外的其他云平台上提供其服务。在某种程度上，[微软](https://thenewstack.io/microsoft-ai-business-agents-will-kill-saas-by-2030/) 将不再向 OpenAI 支付收入分成。

但 OpenAI 证实，从 OpenAI 到微软的收入分成支付“将持续到 2030 年，且独立于 OpenAI 的技术进步”，比例保持不变，但设有总额上限。

## 谁是最终赢家？

现在必须进行的考量是——考虑到两家组织规模的不同，不一定是在同等基础上——在最终博弈中谁获益最多？

OpenAI 最终将寻求与其他云巨头合作，以扩大其企业覆盖范围。同样（或者说作为回报），雷德蒙德（微软总部）希望开发自己的 AI 模型，并推动其 [Microsoft 365 Copilot 商业版](https://www.microsoft.com/en-us/microsoft-365-copilot/business) 产品的增长。

微软在 2019 年向 OpenAI 投入了 10 亿美元（并因此成为其主要云合作伙伴），并 [在当时表示两家公司将](https://news.microsoft.com/source/2019/07/22/openai-forms-exclusive-computing-partnership-with-microsoft-to-build-new-azure-ai-supercomputing-technologies/)：“专注于在 Azure 中构建规模空前的计算平台，该平台将训练并运行日益先进的 AI 模型，包含基于微软超算技术的硬件技术，并坚持两家公司在道德和信任方面的共同原则。”

但那是过去，现在情况不同了。微软将寻求释放其与其他 AI 平台专家合作的能力，其中最明显的包括 [Anthropic](https://thenewstack.io/anthropic-claude-mythos-cybersecurity/) 和 [Google](https://thenewstack.io/googles-gemini-3-1-pro-is-mostly-great/)。

该公司还将寻求减轻与被 [报道称](https://www.forbes.com/sites/aliciapark/2026/04/27/openai-and-microsoft-end-exclusive-partnership-and-revenue-sharing/) “日益紧张”的关系中的某些束缚。这种紧张局势很大程度上源于 OpenAI 相对于整个微软 Azure 体系不断增长的数据中心需求。

> “我们与微软的伙伴关系是我们成功的基石。但它也限制了我们在企业所在地与他们会合的能力——对许多企业来说，那就是 Bedrock。” —— OpenAI 的 Denise Dresser

本月 [CNBC 报道](https://www.cnbc.com/2026/04/13/openai-touts-amazon-alliance-in-memo-microsoft-limited-our-ability.html) 了来自 OpenAI 首席营收官 [Denise Holland Dresser](https://www.linkedin.com/in/denisedresser/) 的一份备忘录，这位营收负责人表示：“我们与微软的伙伴关系是我们成功的基石。但它也限制了我们在企业所在地与他们会合的能力——对许多企业来说，那就是 Bedrock（亚马逊的 AI 服务）。”

## OpenAI 可以在任何云上运行？

目前，微软仍然是 OpenAI 的主要云合作伙伴，这意味着 OpenAI 的产品将首先在 Azure 上发布。OpenAI 的限制条款指出，情况确实如此，“除非微软不能或选择不提供必要的功能。OpenAI 现在可以通过任何云服务商向客户提供其所有产品。”

博士后研究员 [Teodora Groza](https://www.linkedin.com/in/teodora-groza/) 在 [斯坦福法学院博客](https://law.stanford.edu/2025/03/21/ai-partnerships-beyond-control-lessons-from-the-openai-microsoft-saga/) 上分析了目前的情况，并恰如其分地指出了这种伙伴关系的本质，即它*不是*大型科技巨头与新兴 AI 公司之间孤立的协作岛屿，而是刻画了整个 AI 领域趋势的缩影。

“从 Anthropic 到 Inflection AI，公众熟知的 AI 公司几乎没有一家未曾获得过来自 Alphabet、Amazon 或微软的投资，”Groza 写道。“在 2023 年，AI 初创公司筹集的资金中有三分之二来自大型科技玩家，这表明后者可能正在接管传统上由风险资本家履行的角色。”

## AWS：智能体 AI 的下一步是什么？

随着 OpenAI 向其他云巨头开放，[AWS 今天发布的 4 月 28 日演示媒体通稿](https://aws.amazon.com/events/whats-next-with-aws/?trk=e66d7a3a-e6d2-4064-9ea8-d357147f85cc&sc_channel=em&mkt_tok=MTEyLVRaTS03NjYAAAGhbxi3hBfSKTIsV6GK5zB1U37kChT_0ljYA3TL8d1hoQTGaRe3L_mW4Vhc85m8v16KVFt59I_YSGEcaqAB96G00Z0T3opxMIF8WN_5U-O3jC_0h3HR8SCsxng) 提到：“观看 AWS 首席执行官 [Matt Garman](https://www.linkedin.com/in/mattgarman/)、亚马逊应用 AI 解决方案高级副总裁 [Colleen Aubrey](https://www.linkedin.com/in/colleen-aubrey-b90816/) 以及 OpenAI 的领导者们进行坦诚对话，探讨智能体 AI 的未来。”

据 AWS 称，参与者可以直接听取来自 AWS 和 OpenAI 领导者的“真实对话”，内容涉及智能体功能对业务和技术领导者的意义、SaaS 应用，以及适应约束条件根本转变的必要性。

> “这对微软来说是一个健康的举动……同样的逻辑对 OpenAI 同样适用。当它与 Anthropic 竞争企业账户时，客户通常强烈倾向于通过已经集成到其安全周界内的 Bedrock 或 Vertex 来调用模型……” —— Zencoder 首席执行官 [Andrew Filev](https://www.linkedin.com/in/filev/)。

总而言之，虽然 OpenAI 希望“为两家公司追求新机遇提供灵活性”，但微软现有的许可现在将变为非排他性的。微软也在进行风险对冲。该公司承诺作为主要股东，将“继续直接参与 OpenAI 的成长”。

[Zencoder](https://www.linkedin.com/in/filev/) 的创始人兼首席执行官告诉 *The New Stack*，模型训练是一场领先地位不断更迭的竞赛。他表示，尽管微软与 OpenAI 的伙伴关系曾帮助 GitHub Copilot 赢得了早期市场份额，但同样的排他性后来却阻碍了它，并加速了份额向 Claude 的流失。

“这对微软来说是一个健康的举动，”Filev 说道。“主要是因为该公司实际上是一个‘三位一体’的模式，即对某些人来说是云提供商，对需要全套模型的群体来说是 AI API 提供商，而作为应用厂商，它也需要同样全套的模型来保持竞争力。同样的逻辑对 OpenAI 同样适用。当它与 Anthropic 竞争企业账户时，客户通常强烈倾向于通过已经集成到其安全周界内的 Bedrock 或 Vertex 来调用模型。”

## 关系，变得复杂了

与此同时，[*金融时报*今年三月](https://www.ft.com/content/e814f4c3-4fb5-4e2e-90a6-470044436b39?syn-25a6b1a6=1) 报道称，微软正在“考虑对亚马逊和 OpenAI 采取法律行动”，原因是涉及一项价值 500 亿美元的交易，该交易可能违反了其与这家 ChatGPT 制造商的独家云合作协议。这场纠纷的核心在于 AWS 是否被允许提供 OpenAI 的新 Frontier 服务——这是一款用于构建、管理和部署具有深度治理能力的自主 AI 智能体的商业产品。

谈到治理（并寻找真正的赢家）并考虑到数据主权，如果 OpenAI 与微软之间的这种分裂能让行业摆脱云巨头的垄断，并对 AI 基础设施的云服务采取更具平台中立性的态度，那么最终获益的可能是用户自己。

这可能是一个无限美好的愿景，但正如名言所说，[保留所有评判](https://www.goodreads.com/quotes/430610-i-m-inclined-to-reserve-all-judgement-a-habit-that-has)（而非 [预留实例](https://azure.microsoft.com/en-us/pricing/offers/reservations/vm-instances)）或许是所有人的最佳路径。
