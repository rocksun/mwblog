英伟达在多大程度上反映了其富有远见的首席执行官 Jenson Huang 的理念？随着 Jenson Huang 对 OpenClaw 的赞赏及后续支持，他迈出了超越企业边界的一大步，接纳了智能体世界中的这个“坏小子”。英伟达究竟在其中扮演什么角色？

*The New Stack* 就此向英伟达开发者技术总监 [Nader Khalil](https://www.linkedin.com/in/naderlikeladder/) 提出了疑问，并探讨了英伟达如何与开发者合作开展智能体 AI 项目。

![](https://cdn.thenewstack.io/media/2026/06/7de4492e-naderkhalil2025-406x350-1.jpg)

Nader Khalil，英伟达开发者技术总监

Nader Khalil 是 [Brev.dev](https://developer.nvidia.com/brev) 的联合创始人，约两年前被英伟达收购。他的公司曾帮助初创企业使用英伟达的 AI 芯片组。他依然对 AI 的可能性感到兴奋，他的活力证明了英伟达正在享受这一时刻。Nader Khalil 见解广博，展现出初创企业对周围变革速度的敏锐洞察。

首先，Nader Khalil 定义了他所认为的智能体是什么。“我准备了一些幻灯片，”他开玩笑道。但这些幻灯片更多是为了整理他在经常被问到的问题上的思路，而非试图讲课。

> “智能体即大模型（LLM）加外壳（harness）……每一次循环都应让我们更接近目标。”

“智能体即大模型加外壳。如果深入思考，它包含两件事：循环（loop）和大模型。显然，你不希望每次循环都做相同的事情。你希望利用大模型的结果。这可能包括推理出要使用的新工具。每一次循环都应让我们更接近目标。”

Nader Khalil 称赞了 OpenAI 的早期倡议。“ChatGPT 在模型之外进行了创新。这不仅仅是因为他们制作了一个伟大的模型；他们还增加了提示词（prompts）。有系统提示词，然后是用户提示词；还有多模态，突然之间，这让我觉得使用大模型的方式非常棒。每个用户都可以受益于 OpenAI 在你使用个人提示词时所编写的系统提示词。”Nader Khalil 继续说道，“然后他们增加了记忆。”

“突然间，我的助手变得非常有用，因为它能记住关于我的事情。ChatGPT 知道我真的很喜欢烧烤。所以当我问问题时，它记得我的烟熏炉是什么，” Nader Khalil 回忆道。“我当时缺少的是文件处理功能。”

![](https://cdn.thenewstack.io/media/2026/06/61be89fe-image-1024x579.png)

当然，故事通过 Cursor 延续到了 Claude。“但这只是外壳。这里的一切都是外壳，”他说。

Nader Khalil 随后谈到了英伟达今天的工作方式。“在这个快速增长的市场中获得产品的途径是依靠技能（skills）。因此有了 CUDA X 库。”

这些是针对 GPU 加速的用例实现，通常用于计算密集型应用。

“所以我们现在构建的每一个产品，都需要具备技能，因为你需要迎合这个不断增长的受众，”他说。

这就是英伟达最初与内部专家合作，并连接其边缘硬件的方式。

## 支持 OpenClaw

对于英伟达“支持” OpenClaw 的说法，Nader Khalil 表示满意。“我们正处于社区的核心，” Nader Khalil 同意道。“顺便说一句，我们通过许多在开源生态系统中非常重要的项目来做到这一点。”

但 OpenClaw 并非普通项目，与之关联可能会被认为风险很大。“我们在公司有几位开发人员全职为 OpenClaw 做出贡献。”在被进一步追问这种关系的性质时，Nader Khalil 表示：“我认为我们只是尝试在任何可以贡献的地方做出贡献。我认为很明显的是，外壳（harnesses）迎来了一个爆发时刻，对吧？”

> “我们在公司有几位开发人员全职为 OpenClaw 做出贡献。”

这确实是一个重要的时刻。“现在正在发生很多变化，我们非常感谢 [Peter Steinberger](https://www.linkedin.com/in/steipete/)、OpenClaw 以及社区创造了围绕智能体和外壳的这一时刻。我们当然希望有所贡献。”

相关地，OpenClaw 项目目前有许多未解决的合并请求（PRs）。事实上，有传言称不再接受新的 PRs。

> “我们看到 Peter 发推文谈论他们遇到的一些问题，我们就卷起袖子，渴望提供帮助。他们允许我们做出贡献，这让我们感到荣幸。”

“你知道，” Nader Khalil 说，“我们看到 Peter 发推文谈论他们遇到的一些问题，我们就卷起袖子，渴望提供帮助。他们允许我们做出贡献，这让我们感到荣幸。”

Nader Khalil 对此进行了更多反思：“你知道代码的金科玉律。阅读代码比编写代码更难。”

考虑到代码量超过 80 万行，这必然是真的。Nader Khalil 继续说道：“不需要处理这种复杂的代码库确实更容易，但现在的每一个成功项目都有同样的问题。通过招募许多智能体来协助编写代码和构建这些 PRs 更为容易。瓶颈在于合并这些 PRs。”以及处理各种谬误。

“OpenClaw 对行业来说是一次重大变革。这是一个巨大的时刻，所有人的目光都聚焦于此。它在几个月内获得的星标数超过了 Linux。开发者非常关心这个项目，因为它具有影响力，所以我认为你会看到如山般的 PRs，对吧？”

> “它在几个月内获得的星标数超过了 Linux……所以我认为你会看到如山般的 PRs，对吧？”

他们对 OpenClaw 的态度显然是接受其问题，就像对待那个在狂欢派对后被警察拘留但本质善良的喧闹朋友一样。

## 蓝图与微波炉

Hermes 是 OpenClaw 之后涌现的较新项目之一（如 [NanoClaw](https://thenewstack.io/nanoclaw-openclaw-agent-security/)），它希望以更安全的方式重现这种成功。英伟达也正在拥抱它，但 Nader Khalil 回过头来解释了英伟达通常如何看待项目。

“所以，NemoClaw 是我们的蓝图（blueprint）。当我们看到令人惊叹的外壳时，我们会尝试找出如何帮助企业采用它们。消费者有时需要运行任何智能体的安全性；然后是模型和外壳。接下来是技能，对吧？你必须给它访问你终端的权限。”

“蓝图”一词在英伟达中带有一点正式含义，意味着构建 AI 智能体和系统的结构。当然，Nader Khalil 需要展示这些与 Nemotron 模型及其他英伟达解决方案的协作。

“有 Hermes 的蓝图，也有 OpenClaw 的蓝图”。它设置了运行时，如果有本地 GPU 则启用策略，并运行模型。

在企业中使用智能体被视为一项重大风险。“有许多阵营，” Nader Khalil 说。“企业内部有一些团队更加担忧。我们有一个名为 OpenShell 的项目，它是我们的安全运行时，我们将与其合作。”

“我们的目标是创造生态系统所需的工具。工业界和企业界的开发者实际上已经在采用智能体。我们一直在为这些受众进行构建。一种方法是构建专门的智能体或子智能体。”

因此，英伟达并不提供某种“大一统”的解决方案，而是融入团队现有的工作流程中。

### 你的微波炉，你的智能体

“思考这个问题的方式就像当你使用一个以前没用过的微波炉时，你必须按很多按钮或花时间琢磨它。但当它是你家里的微波炉时，你只需‘叮，叮。完成。’对吧？”

“因此，每个企业中的每个行业都将构建这些专门的智能体，许多行业已经这样做了。英伟达已经在与 CrowdStrike、Cadence、Palantir 等众多公司合作。”

## 未来属于智能体

Nader Khalil 相信，人们对长时间运行的智能体的担忧正在慢慢消退。这引出了最后一个问题：英伟达是打算留在可能有恶龙的公海中，还是成为开发者可以工作的平静港湾？

“所以我们的方法是：我们可以帮助谁，以及如何帮助？” Nader Khalil 毫不畏惧。“拐点在几个月前就已经发生了，所以我们问自己能做些什么来迎接所有这些技术。”在这里，Nader Khalil 将他自己——在某种程度上也是英伟达——的未来与新兴开发者联系在一起。

“有些人会很快适应，而有些人不会；我们注意到，如果你看采用曲线，许多人还没有经历过这一点。所以还有很多工作要做，以确保我们能安全地交付这些技术。”
