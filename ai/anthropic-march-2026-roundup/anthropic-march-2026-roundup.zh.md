## Anthropic 的三月简直荒谬

我从事科技创业公司和项目报道已有 20 年，我真的不记得有哪家公司在二月和三月像 Anthropic 这样发布了如此多的内容。几乎每隔一天，Claude 就会通过新功能和新能力进行升级。

以下只是主要发布：

*   [**Claude Opus 4.6**](https://techcrunch.com/2026/02/05/anthropic-releases-opus-4-6-with-new-agent-teams/) 于 2 月 5 日发布，具有 1M token 上下文窗口和 128K token 输出——这是 Anthropic 迄今为止最强大的模型，*The New Stack* [称其为企业领域的变革性一步。](https://thenewstack.io/anthropics-opus-4-6-is-a-step-change-for-the-enterprise/)
*   [**Claude Sonnet 4.6**](https://www.anthropic.com/news/claude-sonnet-4-6) 于 2 月 17 日紧随其后，成为免费和专业计划的默认模型，在 Sonnet 4.5 的定价（每百万 token $3/$15）下，编码、计算机使用和智能体规划方面都得到了升级。
*   [**交互式可视化**](https://claude.com/blog/claude-builds-visuals) 于 3 月 12 日推出——Claude 现在可以使用 HTML 和 SVG 在线构建图表、示意图和交互式小部件，并推广到[所有计划类型，包括免费计划](https://thenewstack.io/anthropics-claude-interactive-visualizations)。
*   [**免费用户的记忆功能**](https://dataconomy.com/2026/03/04/anthropic-makes-claude-memory-feature-free-for-all-users/) 于 3 月 2 日上线，完成了为期八个月的推广，并添加了 ChatGPT/Gemini 导入工具，使切换更简单。
*   [**Claude Marketplace**](https://venturebeat.com/technology/anthropic-launches-claude-marketplace-giving-enterprises-access-to-claude) 于 3 月 6 日上线，允许企业将现有的 Anthropic 支出用于 Replit, GitLab, Harvey, Snowflake 和 Lovable 等公司提供的 Claude 驱动工具——Anthropic 不收取任何分成。
*   [**Claude Code 多智能体评审**](https://thenewstack.io/anthropic-l***unches-a-multi-agent-code-review-tool-for-claude-code/) 于 3 月 9 日推出，调度并行智能体在人工评审人员查看代码之前捕获错误——现在 54% 的拉取请求（PRs）会收到实质性评论，高于之前的 16%。
*   [**Excel 和 PowerPoint 集成**](https://venturebeat.com/orchestration/anthropic-gives-claude-shared-context-across-microsoft-excel-and-powerpoint/) 于 3 月 11 日在这两个应用程序之间实现了共享上下文，通过 Bedrock, Vertex AI 和 Foundry 提供可重用技能和 LLM 网关支持。
*   [**Claude 合作伙伴网络**](https://www.anthropic.com/news/claude-partner-network) 于 3 月 12 日正式成立，承诺投入 1 亿美元——Accenture 正在培训 3 万名专业人士使用 Claude，Cognizant 向其全部 35 万名员工开放了访问权限。
*   [**1M token 上下文**](https://blockchain.news/ainews/anthropic-claude-opus-4-6-and-sonnet-4-6-launch-1m-token-context-at-standard-pricing-business-impact-and-2026-analysis) 以标准定价于 3 月 14 日正式上线（GA），适用于 Opus 4.6 和 Sonnet 4.6，取消了之前超过 200K token 的 2 倍输入附加费。
*   [**Claude Dispatch**](https://support.claude.com/en/articles/13947068-assign-tasks-to-claude-from-anywhere-in-cowork) 于 3 月 17 日作为 Cowork 中的持久智能体线程推出——从手机分配任务，当 Claude 在 Mac 上完成任务时收到推送通知。
*   [**Claude Code 登陆网页和移动端**](https://thenewstack.io/anthropics-claude-code-comes-to-web-and-mobile/)，让开发者可以在 Anthropic 管理的实例上，通过 Claude.ai 启动并行编码工作流程。

除了这些头条功能之外，还有源源不断的小更新：Claude Code 在过去一周的五个点版本中，仅在 Windows 上就增加了 PowerShell 支持、转录搜索、MCP 重复数据删除和空闲返回提示。Cowork 获得了插件支持和文件管理改进。*The New Stack* 一直在密切关注。如果你想跟上进度，我建议从[我们对多智能体代码评审发布](https://thenewstack.io/anthropic-launches-a-multi-agent-code-review-tool-for-claude-code/)和 Claude Code [扩展到网页和移动端](https://thenewstack.io/anthropics-claude-code-comes-to-web-and-mobile/)的报道开始。

## 然而，服务器呢

快速发布也有其弊端：Anthropic 的基础设施似乎难以跟上。仅在三月份，Claude 就至少宕机了五次，其中包括本周两次长时间的中断。在我周五早上撰写此文时，Opus 4.6 正在经历高错误率，尽管 Sonnet 4.6 似乎运行正常。

这并非 Anthropic 独有。OpenAI, Google 和其他公司也都面临着类似的使用量激增带来的增长烦恼。但当你像 Anthropic 现在这样积极发布时，问题会更加明显。你可以拥有世界上最好的模型，但如果开发者无法依赖它的可用性，他们就会构建备用方案，其中一些备用方案会变得永久。可靠性是一种产品功能，而现在这可能是 Anthropic 最需要推出的功能。

## Claude Mythos 意外泄露

周四，*Fortune* 爆料称 [Anthropic 正在测试一款新模型](https://fortune.com/2026/03/26/anthropic-says-testing-mythos-powerful-new-ai-model-after-data-leak-reveals-its-existence-step-change-in-capabilities/)，名为 Claude Mythos——内部代号为“Capybara”——这是一款下一代模型，该公司称其能力实现了“飞跃”，是“我们迄今为止构建的最强大的模型”。

此次泄露并非计划之中。Anthropic 由于公司在 CMS 配置过程中所谓的“人为错误”，意外地将大约 3,000 份未发布资产（其中包括一份博客文章草稿）留在了可公开访问的数据缓存中。安全研究人员发现了它。*Fortune* 在 Anthropic 将其锁定之前进行了审查。

其宣称的能力非常显著：与 Opus 4.6 相比，在编码、学术推理和网络安全基准测试中得分大幅提高。Anthropic 表示该模型“在网络能力方面遥遥领先于任何其他 AI 模型”，并补充说它“预示着即将到来的一波模型，这些模型可以以远远超过防御者努力的方式利用漏洞”。这是 Anthropic 在警告其自己的模型。该公司表示正在与一小部分早期访问客户测试 Claude Mythos，并“慎重”发布。

在三月份的功能狂潮、可靠性问题以及现在蓄势待发的下一代模型之间，Anthropic 目前的故事就是一家公司试图同时以三种不同的速度运行。

## MCP 突破 9700 万安装量

Anthropic 的 Model Context Protocol 在 [3 月份达到了 9700 万月度 SDK 下载量](https://www.digitalapplied.com/blog/mcp-97-million-downloads-model-context-protocol-mainstream/)。这比 2025 年 11 月发布时的约 200 万增长了 4,750%，在 16 个月内。该生态系统现在包括超过 5,800 个社区和企业服务器，涵盖数据库、CRMs、云提供商、开发工具等。当 OpenAI 去年承诺支持 MCP 时，它就不再是 Anthropic 的协议，而是成为了行业的协议。Anthropic 在 12 月正式确立了这一转变，[将 MCP 捐赠给了由 Block 和 OpenAI 共同创立的 Agentic AI Foundation](https://thenewstack.io/anthropic-donates-the-mcp-protocol-to-the-agentic-ai-foundation/)，该基金会隶属于 Linux Foundation。

*The New Stack* 一直密切关注 MCP 的发展。Richard MacManus 写了一篇关于 [MCP 为何成功](https://thenewstack.io/why-the-model-context-protocol-won/)的好文章，我们对 [2026 年路线图](https://thenewstack.io/model-context-protocol-roadmap-2026/)的报道深入探讨了维护者现在正试图弥补的生产就绪性差距——认证、可观测性、大规模服务器管理。该协议显然正在赢得采用，但正如我们的一篇文章所说，MCP 与生产之间仍然存在[一道陡峭的山峰。](https://thenewstack.io/model-context-protocol-evolution/) 对于构建智能体工作流的开发者来说，MCP 是标准。现在的问题是，其周围的工具能否跟上需求。

如果你想和我们一起深入探讨，[我们将在 4 月 2 日至 3 日在纽约市参加 MCP 开发者峰会](https://events.linuxfoundation.org/mcp-dev-summit-north-america/)。我的好朋友 Alex Wilhelm，他出版了出色的时事通讯 *Cautious Optimism*，将在那里为 *The New Stack* 进行采访。

## AI 沙皇卸任

David Sacks 周四告诉 *Bloomberg*，[他已用完 130 天](https://www.cnbc.com/2026/03/26/david-sacks-trump-crypto-ai-czar.html/)作为特别政府雇员的任期，并辞去 Trump 的 AI 和加密沙皇职务。他将与 Michael Kratsios 共同担任总统科学技术顾问委员会（PCAST）的联合主席。这使得 Sacks 拥有更广泛的职责范围，但直接权力可能大大减少。

作为 AI 沙皇，Sacks 与 Trump 有直接联系，并参与政策制定。作为 PCAST 联合主席，他将提出建议。[Axios 报道](https://www.axios.com/2026/03/26/trump-white-house-david-sacks-ai-czar-policy-influence)白宫不打算任命新的 AI 沙皇，这意味着华盛顿最引人注目的 AI 政策职位空缺，且没有替代人选。

在离职前，Sacks 告诉 *Bloomberg*，[国会可能在数月内通过两党 AI 立法](https://www.bloomberg.com/news/articles/2026/03/26/congress-could-pass-ai-standard-in-months-key-trump-aide-says/)，其框架将儿童安全措施与联邦优先于州 AI 法律相结合。“我们得到了国会山的非常好的反响，”Sacks 说。“这是一个我认为我们愿意并乐于与民主党合作的领域。” 这种乐观情绪在他离开日常工作后能否持续下去，这是个问题。AI 政策可能是目前少数真正两党合作的问题，但没有倡导者的两党势头往往会消退。

对于采用 AI 的组织来说，监管格局很重要。联邦优先于州法律将大大简化合规性。零散的州法规是企业在美国各地部署 AI 的最大摩擦点之一。如果 Sacks 的预测没错，法案将在数月内出台，那值得关注。
