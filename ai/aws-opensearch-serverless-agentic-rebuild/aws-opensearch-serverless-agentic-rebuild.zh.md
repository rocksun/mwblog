AWS 周四推出了其所谓的托管搜索和向量引擎的近乎完全重构的版本，旨在更好地满足智能体时代（agentic age）的需求。

新一代 [Amazon OpenSearch Serverless](https://aws.amazon.com/opensearch-service/features/serverless/) 在空闲时可缩容至零，并且与在峰值容量下运行的预置集群相比，旨在降低高达 60% 的成本。

AI 智能体的使用模式通常具有突发性且伴随着长时间的空闲，这基本上打破了 AWS 最初用于 OpenSearch Serverless 的原始无服务器架构的假设。

> “其中约 97% 是由托管服务团队的工程师从头开始构建的。”

今年 2 月成为 AWS OpenSearch 总总经理的 [Tia White](https://www.linkedin.com/in/tia-white-tech-transformation/) 告诉 *The New Stack*：“绝大部分都是大规模的重构。其中约 97% 是由托管服务团队的工程师从头开始构建的。虽然有一些部分已经通过开源仓库提供，但任何真正新颖或属于核心知识产权（IP）的东西，我们都不会通过该开源项目公开。”

## “瑞士军刀”问题

White 解释说，最大的架构变化是存储和计算的分离，OpenSearch 构建在一个新的专有存储层之上。“集合（Collections）可以真正缩减到零，这意味着如果您的资源未激活，您无需支付任何费用，”White 解释道，“然后，由于突发的工作负载，它们可以在几秒钟内重新启动以满足智能体的需求。我们不希望出现冷启动问题。”

她还指出，该服务的自动弹性伸缩速度比上一代快了 20 倍，并且在发布时就已支持搜索和向量集合类型。所有这些都根据用于索引、搜索和 GPU 加速的 OpenSearch 计算单元（OpenSearch Compute Unit）进行计费。此外，本次发布还包括与 Vercel 和 AWS 自家 Kiro IDE 的原生集成，以及一套 OpenSearch 智能体技能（OpenSearch Agent Skills），使开发人员能够使用他们喜爱的工具，包括 Claude Code 和 Cursor。

White 表示，与峰值容量下运行相比，60% 的成本节约来自两个方面：具有压缩功能的新专有存储层，以及能够在一两秒内当流量下降时激进地降低容量的自动伸缩器。

“由于我们能够预测您的需求，并且能够以非常快速的方式提供并向下扩容，您将自动省钱，”White 说。

White 还坦言了为什么 OpenSearch 需要做出这些改变。“此前，OpenSearch 一直充当着瑞士军刀的角色，是大杂烩，”她说，“我们去年甚至试图转向 SIEM [安全信息和事件管理]。”那次弯路并没有坚持下来。新的定位将 OpenSearch 闻名遐迩的传统搜索与日志分析相结合，但完全围绕智能体工作负载进行了重构。

## 即将推出：智能体内存、日志分析以及适用于搜索工作负载的推理模型

这一围绕两大支柱的重新聚焦，伴随着 White 在采访中预览的路线图。

针对智能体的长期记忆功能计划于 2026 年下半年推出，且从第一天起就内置了评估和治理。正如 White 描述该设计问题时所说：“评估（你可以说它是一个治理方面）是一门艺术也是一门科学。关于什么是好的、应该存储什么、应该清除什么的评估方法——这是一个持续的反馈循环。”

> “为我们的客户构建一个智能体优先（agentic-first）的平台，这些都是我们明白必须在第一天就提供的东西。它不能是事后才想到的，也不能是附加组件。”

她说这些护栏不能事后修补。“为我们的客户构建一个智能体优先的平台，这些都是我们明白必须在第一天就提供的东西。它不能是事后才想到的，也不能是附加组件。”该公司还专注于围绕知识图谱和语义层构建 OpenSearch Serverless 的功能，以及 White 所描述的“针对特定搜索工作负载的高级推理模型”。

White 表示，将于 6 月推出一个重大的日志分析产品。这将使 AWS 重新回到目前由 Datadog、Splunk 和 Grafana 主导的市场。随后将在 AWS 纽约峰会上推出 TIMESERIES（时间序列）集合类型，将 OpenSearch Serverless 扩展到更多可观测性（observability）工作负载。

“最终，当精度达到要求、Token 得到优化以及所有这些条件成熟时，你可能会问 LLM 是否可以取代像 OpenSearch 这样的工具，”White 问道。但在 AWS 看来，答案是 OpenSearch Serverless（以及整个 OpenSearch）将成为 LLM 调用的至关重要的语义层——而不是被 LLM 所取代。
