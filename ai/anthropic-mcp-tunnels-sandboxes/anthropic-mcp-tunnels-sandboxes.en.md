Anthropic used its first developer conference, held outside the US in London on Tuesday, to announce new capabilities for [Claude Managed Agents](https://thenewstack.io/with-claude-managed-agents-anthropic-wants-to-run-your-ai-agents-for-you/), the company’s hosted and secure infrastructure layer for running AI agents.

Showcased at [Code with Claude](https://claude.com/code-with-claude), Anthropic added a public beta of its self-hosted sandboxes and a research preview of MCP tunnels. Delivered as updates that address the more pressing needs of AI software engineers today, the team has explained the mechanics behind these functions and the rationale for bringing them to the fore now.

## Anthropic sandboxes, where agents execute

Traditionally used for isolated testing and code experimentation, sandboxes play a similar isolation role in the AI model universe; they exist at this level to protect internal company networks from rogue scripts generated by agents and to shield against those scripts leaking outward to third-party connections.

Putting [data privacy](https://thenewstack.io/llms-and-data-privacy-navigating-the-new-frontiers-of-ai/) compliance, security, and runtime control in the hands of the organization using it, this sandbox is where an agent executes tools and runs on the customer’s own infrastructure or on infrastructure provided by managed service providers such as Cloudflare, Daytona, Modal, or Vercel.

While tool execution moves to the customer’s environment in this scenario, the agent loop (the continuous execution cycle in which an AI service perceives input, reasons, and engages in orchestration, context management, and error recovery) remains on Anthropic’s side.

## MCP tunnels, a lightweight gateway

With MCP acting as the not-quite-de facto (but almost, given its adoption [by the Linux Foundation](https://thenewstack.io/agentic-ai-foundation-launch/)) interconnectivity protocol for the current age of agentic software development, its more managed function in MCP tunnels enables agents to connect to MCP servers inside a private network without exposing them to the public Internet.

Anthropic’s research preview of MCP tunnels is described by the company as a lightweight gateway deployed by the customer that can make a “single outbound connection”, with that connection managed from workspace settings in the Claude Console, as directed by system administrators.

Neither self-hosted sandboxes nor MCP tunnels require changes to existing Claude Managed Agents integrations. Moving between Anthropic’s infrastructure and a customer’s own infrastructure layer is performed via a configuration change, likely applied to exchange cloud-managed API tokens for local authentication keys and to update network routing parameters.

> —Ryan Chang, AI engineering builder, Clay.

## Customers clarify Claude Managed Agents use cases

B2B data platform and orchestration-layer company [Clay](https://www.clay.com/) is using Claude Managed Agents to power its engineering agent, [Sculptor](https://www.clay.com/sculptor), an AI-powered conversational co-pilot. Sculptor builds, tests, and monitors go-to-market (GTM) workflows autonomously on managed agents and Daytona.

The concept is: the power of a local agent with the reliability of the cloud.

[Ryan Chang](https://www.linkedin.com/in/ryancwc/?skipRedirect=true), an AI engineering builder at Clay, has explained that when building Sculptor, the Clay team wanted to give it a more flexible and powerful way to take action than just using tools in a loop.

“Claude Managed Agents let us replicate the power of a local agent with the reliability, versioning, and background execution of a cloud agent,” Chang said. “Running it with our sandboxes, like Daytona, gives us control over the filesystem, so we can mount external file stores and install packages on the fly.”

## Rogo banks on Claude Managed Agents

Financial sector enterprise AI company [Rogo](https://rogo.ai/) is currently developing an institutional finance analyst service for investors and bankers. The organization uses Claude Managed Agents for the reasoning layer and Vercel as the secure execution layer for its proprietary data.

[Strib Walker](https://www.linkedin.com/in/strib-walker-560b56145/), head of product at Rogo, has explained that Claude Managed Agents meets his company’s requirement for agent-loop functionality. The company also subsequently uses Vercel’s sandboxes to provide an environment it can configure for its workloads.

“This gives us the option to leverage best-in-class infrastructure while we focus on what compounds for a financial AI platform: depth and breadth of tools and data, and a product surface built for how investors and bankers actually work,” said Walker.

## Core primitives remain the same

Both self-hosted sandboxes and MCP tunnels work within the same core primitives (the fundamental AI model architecture building blocks, including tokens, weights, and layers) supported by Claude Managed Agents. These updates may be welcomed as significant developments, especially given that Claude Managed Agents was only [launched on April 8](https://thenewstack.io/with-claude-managed-agents-anthropic-wants-to-run-your-ai-agents-for-you/) this year.

Taken alongside the addition of built-in memory and the Claude Platform on AWS becoming generally available, Anthropic has said that these launches have “brought together” the agent loop, memory across sessions, and now an execution and connectivity model that fits enterprise infrastructure.
