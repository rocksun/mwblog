Thomas Dohmke 去年[辞去了 GitHub 首席执行官的职务](https://thenewstack.io/github-loses-its-ceo-and-independence/)，再次成为一名创始人。本周三，他发布了一个分布式 Git 网络的预览版，旨在防止大规模 AI 编程代理群体压垮中央服务器——这一举措可能很快会直接与 GitHub 的核心服务展开竞争。

Dohmke 在 GitHub 之后创立的初创公司 [Entire](https://entire.io/) 于本周三发布了该预览版（目前处于候补名单阶段）。通过此功能，开发人员可以一键将现有的 GitHub 仓库镜像到 Entire 的基础设施上。

“在代理时代，中心化的 Git 托管已成为一个根本性的制约因素，因为数十亿代理和开发人员同时访问中央服务器所带来的压力，表现为速率限制、高延迟，甚至是服务中断，”Dohmke 在今天的公告中表示。“今天，我们开始让 Git 回归其最初的承诺，建立一个由全球互联节点组成的分布式网络，并很快将实现完全去中心化和开源。通过这样做，我们使任何开发人员或代理都能在区域内托管其代码，在靠近其操作的地方进行推送、拉取和克隆，快速且无瓶颈，同时仍然是全球协作网络的一部分。”

Entire 强调，代码的关键在于仍然保留在 GitHub 上，但编程代理可以与 Entire 的镜像协作，正如该公司所言，“构建时不受速率限制”。

Entire 的镜像旨在吸收代理群体可能产生的持续流量。毕竟，这种流量是 GitHub 近期在压力下经常宕机的部分原因，这也为像 Entire 这样的初创公司提供了机会。

Dohmke 在接受 *The New Stack* 采访时表示，中心化 Git 托管现在已经成为“一个根本性的制约因素”，因为数十亿的代理和开发操作汇集在同一台服务器上，表现为速率限制、延迟和中断。

鉴于 GitHub 最近的可用性问题，初创公司试图进入这一领域并不令人惊讶。Entire 就是其中之一，且具有深厚的背景。不过，Cursor 也在六月宣布了 Origin，这是其专门为大规模代理并行克隆和提交单一存储库而重新构建的 Git 工具。

Entire 首先在美国、欧盟和澳大利亚开启活跃区域，但团队表示，随着首批区域的建立，他们很快会增加更多区域。

## ‘将 Git 视为数据库’

为了构建其网络，Entire 重写了 Git 的服务器部分。GitHub、GitLab 和 Bitbucket 都是对 Git 二进制文件的服务器端进行包装，并围绕它构建基础设施。Entire 则是从零开始。

“我们确实将 Git 视为数据库，”Dohmke 说。他解释说，开源 Git 项目分为两半：代理用于与存储库通信的客户端，以及主机运行用于管理存储的服务器。大多数公司会选择在现有的服务器上构建，但“我们决定不走那条路，而是实现了我们自己的 Git 后端。”

当然，这只有在 Entire 的版本比现有的 Git 服务器具有显著更好的性能时才有意义。Entire 表示，其基准测试显示该网络能够达到每小时持续 570,000 次克隆、每秒 586 次推送，以及每秒约 470 次组合克隆和推送操作的性能。

Dohmke 表示，推送到原生的 Entire 分支的速度最高可比推送到 GitHub 快 25 倍。

Entire 表示将开源其 Git 后端和基准测试套件。

## 基础层，现已成真

当 Dohmke [二月份](https://thenewstack.io/thomas-dohmke-interview-entire/)首次向 *The New Stack* 描述 Entire 的计划时，他描述了一个三层平台，底部是一个与 Git 兼容的数据库，中间是一个语义推理层，顶部是一个界面。即使在当时，他也表示，该数据库不像中心化的 Git 托管，可以是一个全球分布式的节点网络。

但在二月份，Dohmke 也表示 Entire 不一定会与 GitHub 竞争，代码存储库将保持在推介的核心地位。

当被问及既然 Entire 托管了 GitHub 存储库的副本，这种说法是否仍然成立时，他称该镜像具有互补性，部分原因是 Entire 可以为企业提供将代码保留在本地区域以满足当地法规的能力。他还指出，GitHub 拥有庞大的生态系统和扩展的功能集。

“我认为买家真正的问题是，从可用性和可靠性的角度来看，我同时拥有这两个产品难道不是更好吗？如果其中一个宕机了——总是会有单点故障和人为错误——那么我还有另一侧的镜像，”他说。“但在交易中，与今天 GitHub 数十亿美元的业务相比，我们肯定会在小得多的规模上争夺支出。”

目前，这使得两者保持互补。Dohmke 认为，GitHub 仍然是“事实来源”或“冷存储”，而工作副本则存在于 Entire 上。但他同时也表示，Entire 将在未来几个月内推出原生存储库，届时将完全不需要底层的 GitHub。所有这些也将开源。

Entire 在二月份完成了 6000 万美元的种子轮融资，当时有 15 名员工。该轮融资由 Felicis 领投，微软的风险投资部门也是支持者之一。公司目前已超过 40 人，目标是到今年年底达到 60 人。

## Git 之外的 Entire：语义记忆层

Entire 正在与 Git 平台并行构建其中间层——语义推理层。

语义层现在可以接入每一个主流的编码代理，包括 Claude Code、Codex、Cursor、Factory AI 和 GitHub Copilot，并将每个会话、提示词和工具调用记录在代码旁边的存储库中。

拥有这些数据对代理很有用，这是该公司推出的第一个核心服务，但现在，它正在基于该历史记录构建更多服务。

例如，该公司正在添加 Entire Blame，它不仅显示是谁最后修改了一行代码，还显示了其背后的代理会话和提示词。此外还有 Entire Review，它会分发多个代理进行意图感知的审查。公司还在添加一项代码和语义搜索功能，允许代理（和开发人员）搜索代码变更及其背后的推理过程。

“会话日志现在是软件开发中第二重要的工件，它们应该与代码一起存在于存储库中，”Dohmke 说。
