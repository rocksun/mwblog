内存已取代算力，成为现代技术团队面临的主要制约因素。硬件架构限制、半导体供应链的不确定性以及软件许可模式的变化，形成了一场完美的风暴，使企业陷入日益严峻的内存受限环境。与此同时，高带宽 AI 负载正超出生产链提供充足内存的能力——而且你的 [AI Token 账单已经开始超过你的薪资](https://www.thestateofbrand.com/news/ai-now-costs-more-than-employees) 支出了。

在过去的一年里，高带宽内存 (HBM) 和动态随机存取存储器 (DRAM) 的成本 [以前所未有的 170% 的速度增长](https://bisi.org.uk/reports/global-ram-shortage-and-price-hikes-causes-consequences-and-market-outlook)，一些 [虚拟化订阅价格甚至翻了一倍多](https://www.cloudtango.net/blog/2025/11/03/vmware-price-increase-what-you-need-to-know/)。

所有这些因素都要求企业从以前“能买就买”的心态转变为数据驱动的优化策略。

幸运的是，AI 不仅仅是问题的一部分。当 AI 被应用于现代虚拟化中的内存经济学时，它就成了解决方案的关键部分。

[Hewlett-Packard Enterprise (HPE)](https://www.hpe.com/us/) 的产品管理总监 [Bharath Ram](https://www.linkedin.com/in/connect2bharath/) 向 *The New Stack* 解释道：“目前存在组件短缺，价格也有所上涨。因此，客户正在寻找节省和优化现有资源占用量的方法，以便在现有设备上运行负载，而不必采购任何新设备。”

通过将重点从吞噬组织的每一点内存转移到优化工作负载及其在多云和混合云环境中的部署，企业不仅可以加速现代化进程，还能将决策周期缩短高达 80%，同时 [降低高达 50% 的成本](https://www.hpe.com/us/en/storage/cloudphysics.html)。继续阅读，了解如何引导您的企业从凭直觉猜测转型为更智能的 IT 模式。

## **技术必须面对其浪费问题**

大多数企业的运营都存在显著的过度配置，这是由于可见性有限和规避风险心理驱动的。

这些公司运行着由于时间推移而效率降低的遗留应用，其中包括大量在没有任何用途的情况下运行的僵尸服务。

除此之外，大多数 AI 工作负载依赖先进的内存技术，这导致芯片制造商将生产重点从 DDR4 转向 DDR5 RAM，进一步减少了 DDR4 的供应。这影响了整个行业，甚至连个人电脑的成本也比去年增加了 15% 到 30%。

再加上波动的 DRAM 定价和更高的核心密度，很明显，即使是非技术领导层也在担心内存效率。科技巨头 Microsoft、Google、Amazon 和 Meta 正在尽可能多地购买 AI 芯片，这引发了整个供应链更多的短缺和价格压力。因此，更多企业正在囤积更多的基础设施和内存。

而且，这种过度购买不仅限于内存。Ram 反思道，公司现在甚至在需要服务器等基础设施之前就开始购买了，“因为成本如此高昂，你今天的报价可能与明天相同基础设施的报价不同。这就是我们目前看到的市场情况，非常动荡。”

但也许一切都会好起来。HPE 估计，目前约有 20% 到 40% 的基础设施处于过度配置状态。这不仅是在这些有限资源中实现增效的机会，也是实现更快、更安全的工作负载的机会。

企业比以往任何时候都更有能力优化现有资源的使用，尤其是在他们去寻找价格显著提高的更多 RAM 之前。

## **一切始于理解**

这种浪费之所以持续存在，是因为企业基础设施是不透明的——没有人真正知道什么数据在做什么，或者哪些服务依赖于它。

阻碍公司进行除了云端“搬迁式迁移”以外任何尝试的因素，通常也是阻碍他们释放内存效率的因素。大多数企业复杂、混合和多云分布式系统的可见性实在太低了。这导致组织十多年来一直在猜测，然后大幅增加预留量。

“这是过度配置和不了解底层基础设施的结合。因为许多人正在进行基于公有云的配置和自服务，你不知道底层基础设施是什么，管理员会利用任何可用的服务功能，”Ram 解释道。“一部分是内存短缺，另一部分是了解已部署的内容并对其进行合理调整（Rightsizing）。”

要打破这些过度配置的坏习惯，任何改变都必须以现实为基础。第一步是收集和分析真实的实际使用数据，使用诸如 [HPE CloudPhysics](https://www.hpe.com/us/en/storage/cloudphysics.html) 之类的工具建立事实基准，将真实的成本驱动因素与多年的假设区分开来。

这使企业能够：

* 了解其虚拟化占用情况和许可风险。
* 查看其工作负载的初始化和效率。
* 在采取行动前识别真正的成本驱动因素。

在实时监控有多少主机拥有多少虚拟机（VM），以及哪些处于开启或关闭状态之前，你无法进行合理调整。

## **预测性而非响应式配置**

一旦企业为其复杂的分布式系统建立了单一事实来源，它就可以探索在何时、何地、以何种方式部署什么。

“像 SAP HANA 这样的应用是高度内存密集型和高度延迟敏感型的。这种算法并不能为了降低成本而在冷热内存分层之间灵活切换，”Ram 解释说，这不能以牺牲应用性能为风险，就像旧电脑内存有限时，一旦用完，计算机就会将程序从内存交换到磁盘运行，导致速度大幅下降。

Ram 认为，现代解决方案的一部分是，公司“可以利用已有的资源进行超分。他们不需要购买任何新内存，”因为单个主机管理的各台虚拟机之间可以使用更好的共享资源。

“例如，一台拥有 64GB 物理内存的主机，分配给各虚拟机的内存总和可能超过物理可用内存，”他解释道。“在实践中，并非所有虚拟机都会同时消耗其全部配额，这允许将未使用的容量根据需要动态地重新分配。”

Ram 表示，内存气球（Memory ballooning）技术并非新鲜事，但正是目前市场迫切需要的。今年夏天发布的 Morpheus 9.0 版本将具备一种更现代的内存超分技术，HPE 解释说，该技术允许管理员在主机上的虚拟机之间超额分配物理内存，从而实现更高的虚拟机密度和更有效的利用。这对于测试和开发环境、虚拟桌面基础设施以及具有多变内存需求的工作负载特别有用。

## **转向架构效率**

最终，当你优化并调整了每一次内存分配后，就该将工作负载迁移到新平台以提高硬件效率了。

“最后一步是提高每台服务器的工作负载密度，尤其是随着按核心计算的软件许可变得越来越昂贵，”Ram 解释道。他表示，实现这一目标的最佳方式是使用带有开源 Hypervisor 的虚拟化解决方案，这可以提高目前的利用率，并帮助组织转向按插槽计费（per-socket licensing）的模式。对于合适的应用，这种现代化改造还可能包括转向容器化部署，而内存内去重则可以减少 RAM 中的冗余数据结构，从而提高内存效率。

“并非所有人都能永远在现有硬件上运行。在某些时候，一些组织将需要迁移到新平台以提高硬件效率。但更高的工作负载密度仍然会带来额外的收益，”他继续说道，特别是在即使是最大的科技公司也在过度购买基础设施、推高成本并收紧容量的时期。

## **按插槽计费节省更多**

如果你确实使用 HPE 的 Morpheus 软件进行了硬件更新，那么你可以开启一种不同的订阅模式，即按插槽或 CPU 授权计费，其中多个核心可以共享一个插槽。一些早期结果表明，这可以节省高达 90% 的成本。
