文章指出，随着公有云成本上涨，盲目上云已不再是最佳选择。企业应重新审视IT架构，根据需求将部分工作负载回迁至本地或私有云，以优化成本并提升管理效率。

> 译自：[Public cloud vs. on-prem: Summit on where each workload belongs](https://thenewstack.io/cloud-costs-private-repatriation/)
> 
> 作者：Alex Wilhelm

距离 [AWS 的发布](https://aws.amazon.com/blogs/aws/amazon_s3/) 已经过去了 20 多年，那场发布会正式拉开了计算和存储从本地机架向云端迁移的序幕。

AWS 以及 Azure 和 Google Cloud 等竞争服务的快速增长，凸显了许多公司如何从掌控自身基础设施转向从超大规模公有云租赁容量。

视频

对于大型供应商而言，公有云已证明是一项令人难以置信的业务。例如，Amazon 的云服务贡献了其第一季度运营利润的 [近 60%](https://ir.aboutamazon.com/news-release/news-release-details/2026/Amazon-com-Announces-First-Quarter-Results/default.aspx)。然而，对于云客户来说，风向可能正在转变。

回想公有云的早期阶段。Azure 和 AWS 为了争夺市场份额，竞相通过 [降价来吸引工作负载](https://awsinsider.net/articles/2014/09/03/microsoft-aws-cloud-price-war.aspx) 进入其中心化计算平台。在随后的几十年里，情况发生了演变。如今，云成本不仅实质性存在而且不断上升，促使一些公司质疑“云优先”是否是最佳发展路径。

云账单不断膨胀，既是因为对超大规模基础设施的使用量增加，也是因为许多客户如今将所有业务都放在云上，而不是仅仅将其用于最适合的场景。

在最新一期的 *The New Stack* 播客中，Summit 的解决方案工程总监 Byron Dill 告诉我们，共享计算和存储在现代 IT 组合中确实占有一席之地，但许多公司最好能细分其工作负载，并将部分工作迁回本地部署。（考虑降低成本以及更简单地管理高风险数据。）

这一论点呼应了我们最近在人工智能领域所见的情况。许多公司迅速采用了 AI 技术，却对随后产生的账单感到惊讶。公有云也是一种类似的“温水煮青蛙”，尽管其周期稍微长一些。

在这两种情况下——AI 和公有云——公司都学到了一个教训：一个最初被推销为降低开支的产品，如果不加谨慎管理，可能会演变成相反的结果。Summit 为企业客户提供托管私有云，该公司认为，一些企业工作负载应该从云端移除并迁回内部。

这需要多少成本？迁移需要多长时间？哪些行业最适合从自己的私有云中受益？我们在本期节目中深入探讨了所有这些问题。
