一份关于 [PHP](https://thenewstack.io/the-herd-is-strong-php-and-its-developer-ecosystem-at-30/) 编程语言现状的新报告显示，由于掌握 PHP 知识的早期开发者减少，正在出现明显的技能缺口。

尽管 PHP 仍然是 Web 应用开发中流行的开源语言，但最近发布的 [Perforce 2026 PHP 景观报告](https://www.perforce.com/resources/zend/2026-php-landscape-report) 指出，组织在寻找和留住资深 PHP 人才方面正面临日益严峻的挑战。

## “15年经验”带来的难题

这项针对全球 700 多名开发者的调查显示，超过一半的受访 PHP 用户表示拥有超过 15 年的该语言使用经验，而只有 15% 的人拥有 5 年或更少的经验。Perforce 官员表示，这表明劳动力正在趋于老龄化，进入该生态系统的新开发者较少。

事实上，招聘已成为 2026 年 PHP 团队面临的首要挑战之一，对于经理和总监来说，这是他们最关心的问题。此外，24% 的受访者将缺乏具备合适技能和经验的人员列为主要的运营挑战。

一位行业分析师对此表示赞同。

Futurum Group 的分析师 [Brad Shimmin](https://www.linkedin.com/in/bradshimmin/) 告诉 *The New Stack*：“对于越来越依赖代理化流程（agentic processes）来生成和维护运行代码的公司来说，我们也看到了技能缺口带来的严重风险。”

## 不仅仅是 PHP 的问题

Perforce Zend 和 OpenLogic 的首席产品经理 [Matthew Weier O’Phinney](https://www.linkedin.com/in/mweierophinney/) 在一份声明中表示：“这不仅仅是 PHP 的问题。这是一个开源界共同的问题。组织依赖 PHP 处理关键任务应用，但随着经验丰富的开发者退休或离职，填补这些专业知识空缺正变得越来越困难。”

在该公司最近的“2026年上半年数据智能、分析和基础设施决策者调查”中，他们发现技术人才严重短缺，在过去六个月中这一比例翻了一番多，达到 10.4%。[Brad Shimmin](https://www.linkedin.com/in/bradshimmin/) 表示，这对于维护代码来说是一个“定时炸弹”。

## 新人减少，债务增加

他说：“手动构建复杂的数据管道，并在多年不可避免的架构演变中维护数千行定制 [SQL](https://thenewstack.io/sql-vs-python-frenemies-of-the-data-world/) 是一项艰巨的工作，工程人才根本不想做这种工作。但当你依赖 AI 来编写和维护这些代码时，不可避免地会开始累积大量的隐形成本和技术债务。随着进入职场的初级开发者越来越少，这个问题的范围和规模只会不断扩大。”

[Brad Shimmin](https://www.linkedin.com/in/bradshimmin/) 补充说，他希望行业继续投资于能够构建和保留机构知识与领域专业知识的工具，以透明、可解释且最终可维护的方式引导系统向前发展。

Forrester Research 的分析师 [Andrew Cornwall](https://www.linkedin.com/in/acornwall/) 告诉 *The New Stack*：“在开发者心目中，PHP 被视为成熟技术，很少有新开发者去学习它。话虽如此，PHP 并不难学。PHP 最初代表‘个人主页（Personal Home Page）’，许多使用它 15 年或更长时间的开发者都是自学成才的。虽然人力资源部门可能很难找到上过 PHP 课程的初级程序员，但公司不难找到开发过 [WordPress](https://thenewstack.io/wordpress-alternatives-stick-with-php-or-pivot-to-javascript/) 主题或为 WooCommerce 站点添加过自定义字段的人。由于 PHP 代码量如此庞大——GitHub 上有 13.9 万个公共仓库——它也是 [AI 生成](https://thenewstack.io/ai-code-generation-trust-and-verify-always/) 的理想选择。”

与此同时，Perforce Zend 的专业服务经理 [Adam Culp](https://www.linkedin.com/in/adamculp/) 在报告中写道，与往年一样，内部 PHP 专业知识的缺口已经进一步扩大。

他写道：“招聘继续成为首要挑战。在更广泛的 OSS（开源软件）生态系统中，获取技术支持和技能人员仍是一个长期存在的问题。再加上受访人群中只有 15% 的人拥有 5 年或更少的 PHP 经验，这一挑战的根源显而易见：新开发者越少意味着招聘池越小，尤其是在需要更替现有职位的时候。”

## 依然驱动着互联网

调查还指出，PHP 继续作为 Web 应用程序和 API 的基础技术。它与 [JavaScript](https://thenewstack.io/30-years-of-javascript-10-milestones-that-changed-the-web/) 并列成为 2026 年使用最广泛的语言，占比达 72%。紧随其后的是 [Python](https://thenewstack.io/what-is-python/) (49%)、Node.js (46%) 和 [Java](https://thenewstack.io/java-at-30-the-genius-behind-the-code-that-changed-tech/) (35%)。

报告还指出，一个值得关注的统计数据是使用 [Go](https://thenewstack.io/introduction-to-go-programming-language/) 语言的受访者比例很高 (21%)，它在多个细分领域进入了前五名，包括成员 20 人及以上的开发团队以及拥有 10 年以上经验的 PHP 开发者。

[Andrew Cornwall](https://www.linkedin.com/in/acornwall/) 告诉 *The New Stack*：“PHP 开发者认为 PHP 是一种流行的开源语言并不奇怪。然而，大型企业往往不会在新项目或大型应用中使用 PHP：报告发现 82% 的 PHP 开发者来自员工人数少于 500 人的公司。”

[Andrew Cornwall](https://www.linkedin.com/in/acornwall/) 提到 WordPress、[WooCommerce](https://woocommerce.com/) 和维基百科都是用 PHP 编写的。最初 PHP 完全专注于服务器端脚本，但现代 PHP 也支持 [CLI](https://thenewstack.io/learn-to-love-the-command-line-interface-with-agentic-llms/)，并能通过 [NativePHP](https://nativephp.com/) 等框架开发桌面应用。由于存在时间很久，PHP 与 SQL 数据库有着极佳的集成性。互联网的很大一部分仍然运行在 PHP 上——包括数百万个电子商务网站——只是大型企业 Web 领域除外。

关于 PHP 应用集成的系统，关系型数据库是首选，占比 92%，其次是 Web API (83%)、文件系统 (70%) 和键值存储 (60%)。MySQL 和 MariaDB 并列成为最常用的数据技术，占比 60%。

至于使用 PHP 编写的主要应用类型，报告显示服务或 API 是最主要的 PHP 应用类型 (80%)，其次是内部业务应用 (70%) 和内容管理 (56%)。

此外，最常用的 PHP 框架是 [Symfony](https://symfony.com/)（占比 47%），其次是 [Laravel](https://laravel.com/) (40%)、[Laminas/Mezzio](https://getlaminas.org/) (8%) 和 [CakePHP](https://cakephp.org/) (5%)。

[Andrew Cornwall](https://www.linkedin.com/in/acornwall/) 表示：“PHP 现在是、将来也将是互联网的重要组成部分。尽管如此，大多数 PHP 开发工作是对现有程序进行细微调整或根据 PHP 新版本进行必要更改，而不是开发重大的新应用。”
