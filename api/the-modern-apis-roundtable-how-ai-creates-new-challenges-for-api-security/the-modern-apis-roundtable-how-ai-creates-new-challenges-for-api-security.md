AI时代API面临发现、安全、标准化、治理挑战。需集中平台、早期安全测试、AI文档、适应性管理工具，以应对API与AI融合带来的机遇与复杂性。

> 译自：[The Modern APIs Roundtable: How AI Creates New Challenges for API Security](https://thenewstack.io/the-modern-apis-roundtable-how-ai-creates-new-challenges-for-api-security/)
> 
> 作者：Adam LaGreca

API是互联网的命脉，实现了应用程序、服务和设备之间的无缝通信。随着AI技术的兴起，API的战略重要性日益凸显，为智能应用程序、大型语言模型（LLMs）和复杂生态系统提供了强大支持。然而，这种快速演进在发现、安全、标准化和治理方面带来了严峻挑战。

我最近有幸主持了一场圆桌讨论，小组成员包括 Joni Klippert（[StackHawk](https://www.stackhawk.com/) 首席执行官）、Rodric Rabbah（[Postman](https://www.postman.com/) 产品主管）和 Alex Drag（[Kong](https://konghq.com/) 产品营销主管）。这些行业领导者强调了在AI时代塑造现代API管理的关键问题和新兴方法。

## API发现和可见性的挑战

随着API在组织环境中激增，维护一个全面且最新的清单变得越来越困难。Kong 的 Alex Drag 详细讨论了在多个网关之间管理API的挑战。与此同时，StackHawk 的 Joni Klippert 指出，安全团队通常只关注总API版图的一小部分，这些盲点使组织面临重大的安全风险。

为解决这一问题，小组强调了拥有一个集中式平台来发现和查看组织内所有API的重要性，包括所有权、安全策略和使用监控等元数据。代码扫描和与现有[开发者工具](https://thenewstack.io/gemini-code-assist-review-code-completions-need-improvement/)的集成等技术有助于提高可见性和控制，使组织能够跟上其不断扩展的API生态系统。

## 复杂生态系统中的主动API安全

从被动安全实践转向主动安全实践也是对话中反复出现的主题。StackHawk 专注于在开发过程中测试API，这强调了尽早识别漏洞的重要性。“安全债”（未经测试或不安全的API累积）的概念对组织构成重大威胁。

Postman 的 Roderick 强调了在部署前持续监控、扫描和保护API的重要性。协调开发者激励措施并提供明确的安全指导可以促进主动行为，从而降低泄露风险并增强整体API弹性。

## 简化文档和标准化

保持API文档的及时性和准确性仍然是一个核心挑战。开发者通常将文档视为一项繁琐的任务，导致出现可能阻碍安全和治理的空白。StackHawk 提倡使用AI驱动的解决方案，从源代码自动生成API规范，从而减少手动工作并提高准确性。

另一个问题是组织内部缺乏标准化。Postman 的 Rodric Rabbah 承认，API网关实现的多样性使这项工作复杂化，但最终得出结论，在快速演进的环境中，维护一个一致、集中的API版图对于治理和安全保障至关重要。

## API货币化和AI的影响

[API日益被视为具有战略意义](https://thenewstack.io/beyond-monitoring-the-urgent-need-for-strategic-api-consumption-management/)的资产，能够通过内部和外部的货币化模式产生收入。AI和LLMs的出现带来了新的[复杂性和机遇——需要更好的治理](https://thenewstack.io/kubecon-keynotes-wrestle-with-ai-governance-complexities/)、安全性和可见性。

Drag 开启了关于[API 管理工具](https://thenewstack.io/api-management/ "API 管理工具")演进的讨论，以支持新兴用例，包括API记分卡和模型组合协议（MCP）。随着API和AI的融合加速，组织必须调整其管理策略，以确保安全性、成本控制和运营敏捷性。

## 结论

圆桌讨论全面描绘了塑造现代API管理的挑战和解决方案。随着API与AI日益紧密地结合，组织需要主动的、集中式的自动化工具来有效地发现、保护和治理其数字资产。

Kong、Postman 和 StackHawk 等供应商正在从不同角度引领潮流，提供创新的[解决当今API](https://thenewstack.io/how-devsecops-teams-should-approach-api-security/)复杂现实的方法。通过采用这些解决方案，组织可以充分释放AI驱动创新的潜力，同时保持安全性、合规性和运营效率。
