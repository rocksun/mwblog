GSMA Open Gateway通过标准化API，将300多家移动网络的核心电信能力整合成一个统一的可编程层，极大简化了开发者访问和利用网络功能的方式。它旨在提供可信赖、生产就绪的API，解决开发者面临的跨运营商复杂性，并增强应用安全性和效率，最终推动电信基础设施向开发者友好型服务转型。

> 译自：[GSMA Open Gateway offers developers one API for 300+ mobile networks](https://thenewstack.io/gsma-open-gateway-developers/)
> 
> 作者：Adrian Bridgwater

开发者在乎协议、标准和规范——但只有一点点。这些并不是让他们夜不能寐的原因。

普通的软件工程师更关心功能性、性能、调试、错误配置以及控制基础设施的复杂性。

如果技术堆栈中的某个组件不符合这些目标，它很少能进入开发者的“待办事项列表”（backlog），即跟踪应用程序功能、增强和修复的战略工具。

这些“实话”可能让作为移动通信行业倡导和游说组织的GSMA领导者感到焦虑，因为他们的[Open Gateway倡议](https://www.gsma.com/solutions-and-impact/gsma-open-gateway/)很难向开发者行家推销。

Open Gateway是一种网络技术，提供一套标准化的API，允许软件工程师跨不同运营商访问和评估电信运营商的能力（例如位置或安全）。

还是不感兴趣？好吧，我们懂。它是电信基础设施，除了电信公司，谁会在乎呢？但如果它将核心电信特性和能力定位为安全堆栈中的可编程层呢？哦，那开发者中心化的电信连接分析和控制即服务听起来就更有趣了。

## 面向真实开发者的电信API

GSMA表示，Open Gateway倡议提供的工具让开发者拥有统一的通用语言，而不是让开发者在全球每个地区的每次安装中都需要“说”500种不同的“电信语言”。

> “GSMA表示，Open Gateway倡议提供的工具让开发者拥有统一的通用语言，而不是让开发者在全球每个地区的每次安装中都需要‘说’500种不同的‘电信语言’。”

GSMA网络负责人兼Open Gateway项目负责人Henry Calvert告诉《The New Stack》，这意味着在伦敦构建的移动应用程序在东京或纽约也能以完全相同的方式运行——开发者无需更改一行代码。

“这正是看待GSMA Open Gateway倡议的视角，”Calvert说。“自三年前公布计划以来，我们现在已获得85个运营商集团旗下300多家移动网络的支持，覆盖全球80%以上的移动连接。

“这些亮眼的数据表明了行业的一致性——但开发者的问题更简单：这是否能转化为通过API暴露出来的可用、生产就绪的能力？”

本周在巴塞罗那举行的世界移动通信大会上，GSMA领导人预测，行业对话将从潜能转向实际实施。重点正从标准和规模转向实际部署、开发者采用和可衡量的影响。

## 来自网络的安全信号

那么，这里提供了哪些核心开发者功能呢？

Calvert提供了一个实际用例：呼叫号码验证和SIM卡互换检测等网络API功能暴露了难以在应用层复制的运营商级别信号。

开发者无需仅仅依靠启发式方法，即可查询网络以验证号码与设备的关系是否完好。从实际角度看，这与理解5G核心架构无关；它完全关乎将更强的信任信号集成到现有的认证或风险引擎中。

开发者可以模拟复杂场景——例如SIM卡被盗或用户在不同基站之间移动——而无需对可能数千部实体手机执行移动设备管理任务。这是一个虚拟世界，他们可以在其中安全地“破坏”事物，以便在现实世界中完美运行。

“这样来看，Open Gateway将电信能力定位为安全堆栈中另一个（我们认为相当吸引人）的可编程层，与身份提供商、设备智能平台和欺诈工具专家的功能并驾齐驱，”Calvert告诉《The New Stack》。“网络API的认知度正在提高，但认知度不等于采用。电信术语、策略模型和网络行为对大多数软件工程师来说仍是陌生的领域。正如任何平台转型一样，开发者体验将是决定性的。”

Open Gateway团队倡导清晰的文档、[CAMARA](https://camaraproject.org/)等开源API仓库，以及对沙盒和生产环境的简单访问。它还偏好主流语言的软件开发工具包（SDK），以加强该倡议所倡导的一致性因素。

> “开发者不应该需要理解每个移动网络的细微差别来构建安全、智能的数字体验。”

为何在这里强调标准化？因为在运营商层面达成一致的电信标准并不总是自动转化为统一的运行时行为。延迟特性、错误处理和服务可用性需要足够可靠，开发者才能在生产环境中信任这些API。

CAMARA源自希腊语“拱形屋顶”（kamāra），象征着多个实体在同一愿景下的联盟，它是Linux基金会的一部分，并与GSMA密切合作。它被称为GSMA Open Gateway倡议的“发动机室”。其使命是定义、开发和测试标准化API，以连接复杂的电信网络和应用程序。

[Linux基金会](https://www.linuxfoundation.org/)网络、边缘和访问首席技术官Ranny Haiby告诉《The New Stack》：“开发者不应该需要理解每个移动网络的细微差别来构建安全、智能的数字体验。”

Haiby说：“由Camara开源项目驱动的GSMA Open Gateway，让现在使用AI工具的开发者能够轻松发现和使用电信网络提供的激动人心的服务。”“我们的目标是通过开放、标准化的API和开发者可以信任的工具，在各个市场提供网络能力。”

## 连接能力现在是可编程的

对GSMA的Calvert来说，更广泛的转变在于，连接能力正被重新定位为可编程资源而非被动基础设施，使API成为电信投资与开发者希望构建的成果之间的桥梁。

“对于构建对延迟敏感的服务、防欺诈注册流程或全新实时体验的开发者来说，这开启了他们以前无法控制的设计可能性。它也符合更广泛的趋势，即自动化、API优先架构和AI辅助网络运营，所有这些都旨在通过软件抽象使日益复杂的基础设施变得可消费。”

Calvert表示，所有这些努力旨在提供预构建的安全密钥，这样开发者就不必构建自己的“锁”。这确保了当应用程序向网络请求数据时，默认情况下以最高级别的隐私和安全来完成。

GSMA表示，下一阶段将取决于运营纪律和真正的开发者参与。如果网络API提供可靠性、清晰度和可衡量的价值，Open Gateway可能成为现代开发者工具包的有意义补充。否则，它就有可能仍然是一个由行业主导的愿景，却从未完全转化为开发者级别的工具。
