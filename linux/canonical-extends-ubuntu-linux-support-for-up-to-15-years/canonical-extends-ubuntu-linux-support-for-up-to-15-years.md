Canonical推出Ubuntu Pro Legacy附加组件，为Ubuntu LTS版本提供15年支持。该服务面向企业，确保旧版系统安全合规。26.04 LTS也将享15年支持，但价格更高。

> 译自：[Canonical Extends Ubuntu Linux Support for up to 15 Years](https://thenewstack.io/canonical-extends-ubuntu-linux-support-for-up-to-15-years/)
> 
> 作者：Steven J. Vaughan-Nichols

所以，你说你想要真正、真正地长期支持你的 Linux？孩子，[Canonical](https://canonical.com/) 为你准备了一份大礼！

有些人真的不想更换他们的企业操作系统。他们是那种在生产环境中[仍在运行 Windows 2003](https://www.reddit.com/r/sysadmin/comments/1dddiz8/who_still_has_win_2003_servers_in_their/?rdt=39796)（没错，真的）或 [CentOS 7](https://ciq.com/blog/one-year-after-eol-centos-is-still-a-force/) 的人。当然，他们冒着风险在早已过时的操作系统上运行服务器。

但是，如果我告诉你，你仍然可以运行旧操作系统并获得全面支持或安全保障呢？这就是 Canonical 正在做的，通过其全新的 [Ubuntu Pro Legacy 附加组件15年支持服务](https://ubuntu.com/pro)。

Canonical 的 Legacy 附加组件现在将所有 Ubuntu 长期支持 (LTS) 版本的支持延长至15年，从2014年的 [Ubuntu 14.04 (Trusty Tahr)](https://thenewstack.io/canonical-extends-support-for-older-ubuntu-linux-distros/) 开始，算起自其最初发布之日起的完整15年。增强的支持建立在标准的五年免费维护、通过 Ubuntu Pro 提供的五年扩展安全维护 (ESM) 的基础上，并通过高级 Legacy 附加组件引入了额外的五年支持。

## 旨在企业领域的扩展支持

这并非 Canonical 首次为其操作系统中偏爱稳定性而非其他一切的用户提供延长支持生命周期。去年，[Ubuntu 将支持延长至12年](https://opensourcewatch.beehiiv.com/p/canonical-unveils-12-years-support-ubuntu-lts)。

这一延长的生命周期特别针对企业和受监管行业，如银行、医疗保健和制造业，在这些领域，重大升级造成的停机时间可能成本高昂或不可行。

在整个15年期间，Ubuntu Pro 用户将获得针对影响基础系统和主要组件的关键、高危和主要中等级别的常见漏洞与暴露 (CVE) 的安全补丁，以及实时内核修补和长期漏洞管理等合规性功能。

然而，与此同时，Canonical 也因在其版本中支持更新的技术而闻名。例如，[Ubuntu](https://ubuntu.com/) 的下一个 LTS 版本 Ubuntu 25.04 将[包含 sudo-rs，一个用内存安全 Rust 编写的 sudo 版本](https://thenewstack.io/why-sudo-rs-brings-modern-memory-safety-to-ubuntu-26-04/)。

然而，正如 Canonical 的 Ubuntu 工程副总裁 Jon Seager 在 LinkedIn 上所写：“在过去的几个月里，我们通过将 Ubuntu 转向采用更现代的工具（sudo-rs、uutils）而吸引了大量关注，我坚信这对于长远发展来说是正确的举措。话虽如此，现实是许多组织在迁移方面遇到困难，并发现自己陷入合规性陷阱，这使得他们[需要长时间运行旧版本。](https://www.linkedin.com/feed/update/urn:li:activity:7394727771010355201/)”

因此，推出了针对旧发行版的这项新计划。

## 扩展支持计划的详细信息

具体来说，这项扩展支持计划将为整个 Ubuntu 基础系统、内核和关键开源组件提供持续安全维护。在此期间，Canonical 将主动扫描、分类和回溯移植关键、高危和部分中等 CVE 到所有受维护的 LTS 版本。

故障排除支持仍是一个可选附加组件。当生产问题出现时，您可以通过此服务联系 Canonical 的支持团队，并与 Ubuntu 专家一起排除故障。

这确实伴随着高昂价格。第10年之后的 Legacy 附加组件将比通常的 [Ubuntu Pro 订阅](https://ubuntu.com/pricing/pro)贵约50%。小型企业可能会觉得这是一个负担，但迁移也并非廉价。然而，寻求真正长期支持的大型企业无疑会将其视为一种经济高效的方式，以确保连续性和安全性，而无需强制升级。

即将发布的 Ubuntu 26.04 LTS (Resolute Raccoon) 定于2026年4月发布，将是首个从发布伊始即提供15年支持路径的新版本——对于付费订阅用户而言，将一直支持到2041年。

尽管这项新的支持计划很长，但它并非目前主要企业级 Linux 发行商提供的最长的。这项殊荣属于 [SUSE](https://www.suse.com/) 及其全新的 [SUSE Linux Enterprise Server (SLES) 16.0 版本](https://www.zdnet.com/article/suse-enterprise-linux-16-is-here-and-its-killer-feature-is-digital-sovereignty/)。最新的 SLES 总共提供16年支持：10年标准支持加6年扩展生命周期支持。然而，与 Canonical 不同，SUSE 对其较旧的主要发行版本仅提供13年支持。
