本文介绍了 Block 将 AI 编程智能体 Goose 移交给 Linux 基金会旗下 AAIF 的原因。通过解决商标权和治理透明度问题，此举旨在消除企业采用障碍并促进生态协同。

> 译自：[Why Block handed Goose to the Linux Foundation](https://thenewstack.io/block-goose-agentic-foundation/)
> 
> 作者：Alex Wilhelm

内部工具可以演变为强大的外部服务。亚马逊云（AWS）就是这样的案例。Block 的 AI 编程智能体 Goose 也是如此，Block 在[向所有员工推广该工具](https://thenewstack.io/how-block-got-12000-employees-using-ai-agents-in-two-months/)后，将其[开源](https://block.xyz/inside/block-open-source-introduces-codename-goose)了。

然而，以许可协议向更广泛的技术社区发布 Goose 并非故事的终点。

*The New Stack* 在一场 MCP 开发者活动中采访了 Manik Surtani。他是新任命的 Agentic AI Foundation 首席技术官兼联合创始人，此前曾担任著名金融科技公司 Block 的开源主管，以深入了解更多信息。

> “Goose 并非完全自由和开放，因为 Block 仍持有其商标，这阻碍了企业的采用。”

Santani 告诉 *The New Stack*，在向公众发布 Goose 并见证其早期的快速普及后，Block 开始遭遇“一些阻力”。Surtani 解释说，团队遇到的最大问题是 Goose 项目在治理上缺乏透明度。由于 Block 仍然拥有其商标，Goose 并非完全自由开放，这限制了企业的采纳。如果 Goose 想要真正展翅高飞，就必须做出改变。

治理问题的答案是将 Goose 移交给基金会。但交给哪一个呢？在与 MCP 团队、Goose 团队以及 Anthropic 公司 Model Context Protocol 协议背后的优秀团队交流后，我们决定携手合作，共同创立 Agentic AI Foundation（AAIF）。如今，AAIF 是更大的 Linux 基金会的一个分支，并且得到了企业界（参见[我们与](https://thenewstack.io/mcp-summit-aws-bedrock/)[AWS 团队](https://thenewstack.io/amazon-aws-mcp-agentic/)关于指导 MCP 发展的交流）和开发者的共同支持。

> “……携这三款工具启动基金会的部分原因是出于权宜之计，希望让该组织尽快运转起来。”

Goose、MCP 和 [Agents.MD](http://agents.md) 构成了 AAIF 的早期核心。但 Surtani 表示，携这三款工具启动基金会的部分原因是出于权宜之计，希望让该组织尽快运作。欲了解更多关于哪些项目可能加入 AAIF 伞下、Goose 最初是如何编写的等内容，请点击播放！
